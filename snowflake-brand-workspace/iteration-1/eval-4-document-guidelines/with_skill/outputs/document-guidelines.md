# Snowflake Brand Guidelines for Documents: Internal vs. Customer-Facing

This guide provides differentiated brand guidance for two document types involving our Snowflake partnership: internal strategy documents and customer-facing proposals. All specifications are drawn from the official Snowflake brand guidelines.

---

## Shared Fundamentals (Both Document Types)

These rules apply universally regardless of audience:

- **Primary brand color**: Snowflake Blue `#29B5E8` -- use the exact hex value, never an approximation.
- **Body text color**: Midnight `#000000`.
- **Body font**: Lato (available via Google Fonts). Use for all body copy, subheadings, descriptions, and form labels.
- **Headline font**: TEXTA (if licensed and available). When unavailable, substitute a geometric sans-serif such as Montserrat or Nunito Sans. Note that this is not an officially sanctioned substitution.
- **Snowflake logo**: Never distort, rotate, recolor, or add effects. Always maintain minimum clear space on all sides equal to the x-height of the logotype. The Snowflake bug (icon) must always include the registration mark.
- **Data visualizations**: Follow the recommended color sequence -- Snowflake Blue, Valencia Orange (`#FF9F36`), Purple Moon (`#7D44CF`), Star Blue (`#71D3DC`), First Light (`#D45B90`), Windy City (`#8A999E`).

## Customer-Facing Proposal

A customer-facing proposal represents both your organization and Snowflake to an external audience. Brand compliance must be strict.

### Color Usage

| Element | Color | Hex |
|---------|-------|-----|
| Headers, dividers, highlights | Snowflake Blue | `#29B5E8` |
| Body text | Midnight | `#000000` |
| CTAs and emphasis text | Snowflake Blue | `#29B5E8` |
| Hover states and depth accents | Mid Blue | `#11567F` |
| Neutral / muted text | Windy City | `#8A999E` |

- Use **primary colors only** for dominant visual elements. Keep secondary and tertiary colors to a supporting role.
- Do not use tertiary palette colors (Iceberg, Ruby Sky, Violet Dawn, Winter) as primary section backgrounds in customer-facing materials unless you are intentionally creating a dark-themed hero section.

### Logo and Co-Branding

- Include the official Snowflake logo using the correct variant for the background:
  - **Snowflake Blue logo** on white or light backgrounds.
  - **White logo** on Snowflake Blue backgrounds or dark/image-heavy backgrounds.
- When placing your company logo alongside the Snowflake logo:
  - All logos must be **center-aligned**.
  - Logos must be **spaced equally**.
  - Logos should appear **optically the same size** (match visual weight, not just pixel dimensions).
- Refer to the partner guidelines PDF in `assets/partner-guidelines/` for approved co-branding layouts (horizontal co-partnership, vertical co-partnership, horizontal multi-partnership, vertical multi-partnership).

### Typography

- Headlines: TEXTA (or approved fallback) in a heavier weight for maximum impact.
- Body: Lato at regular (400) weight for readability. Use bold (700) sparingly for emphasis.
- Button text (if applicable): TEXTA Heavy, ALL CAPS, max 13 characters.

### Presentation Slides

- Start from the **official 2025 PowerPoint template** (`assets/Snowflake_Template_2025.pptx.zip`).
- Do not create custom slide masters that deviate from the template's color scheme or layout grid.

### Compliance Checklist (Customer-Facing)

- [ ] Primary accent color is exactly Snowflake Blue (`#29B5E8`)
- [ ] Logo uses the correct variant for its background and has proper clear space
- [ ] Body text uses Lato; headlines use TEXTA or an approved geometric sans-serif fallback
- [ ] Partner logos are center-aligned and optically equal in size
- [ ] No unapproved color modifications to the Snowflake logo
- [ ] CTAs and emphasis text use Snowflake Blue
- [ ] Any buttons follow the pill-shape spec (80px border-radius for desktop)
- [ ] Button text is ALL CAPS, max 13 characters
- [ ] Presentation slides are built from the 2025 template

## Internal Strategy Document

Internal documents still represent the partnership to colleagues and stakeholders, but you have more flexibility in layout and color application.

### Color Usage

| Element | Color | Hex |
|---------|-------|-----|
| Primary headings and key highlights | Snowflake Blue | `#29B5E8` |
| Body text | Midnight | `#000000` |
| Section backgrounds, sidebars | Iceberg or Winter | `#003545` / `#24323D` |
| Visual hierarchy accents | Any secondary color | See palette below |
| Neutral borders and muted text | Windy City | `#8A999E` |

- **Secondary colors can be used more liberally** for visual hierarchy -- for example, using Valencia Orange (`#FF9F36`) for callout boxes, Purple Moon (`#7D44CF`) for category labels, or First Light (`#D45B90`) for decorative accents.
- **Tertiary colors work well** for section backgrounds and sidebar elements. Iceberg (`#003545`) and Winter (`#24323D`) are particularly effective for dark-themed sections that set off content blocks.

### Logo

- Including the Snowflake logo is optional for purely internal documents but recommended for strategic alignment materials.
- If included, follow the same variant and clear-space rules as customer-facing documents.
- Co-branding layout rules still apply if partner logos appear.

### Typography

- Same font rules apply (Lato for body, TEXTA or fallback for headlines), but you have more latitude with weight and size choices for internal readability.
- For dense strategy documents, consider using Lato Light (300) for supporting annotations and Lato Bold (700) for section headers to create a clear hierarchy without relying solely on color.

### Layout Flexibility

- You may use multi-column layouts, sidebar navigation, or unconventional grid structures that would not be appropriate in a customer-facing deliverable.
- Tertiary palette dark backgrounds are well-suited for executive summary sections or key-metrics dashboards within the document.
- Data visualizations should still follow the official color sequence to maintain consistency across internal and external reporting.

### Compliance Checklist (Internal)

- [ ] Color palette uses official Snowflake hex values (no approximations)
- [ ] If the Snowflake logo appears, it uses the correct variant and has proper clear space
- [ ] Body text uses Lato; headlines use TEXTA or fallback
- [ ] Data visualizations follow the recommended color sequence
- [ ] Dark sections use tertiary palette colors (Iceberg, Winter, Violet Dawn), not arbitrary dark tones
- [ ] Secondary colors are used intentionally for hierarchy, not randomly
